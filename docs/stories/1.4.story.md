# Story 1.4: Ghi dữ liệu vào TimescaleDB

## Status
Draft

## Story
**As a** hệ thống,
**I want** Go client đồng thời ghi dữ liệu tick/nến nhận được vào TimescaleDB,
**so that** lưu trữ lịch sử dữ liệu nóng ngắn hạn cho việc phân tích theo cửa sổ thời gian.

## Acceptance Criteria
1. Schema cho bảng dữ liệu thị trường (với hypertable) được tạo trong TimescaleDB.
2. Go client kết nối thành công tới database TimescaleDB.
3. Mỗi điểm dữ liệu nhận được được chèn (insert) như một hàng mới vào bảng tương ứng trong TimescaleDB.

### TimescaleDB Schema
[Source: docs/architecture/phn-8-s-database-database-schema.md]

**Table: ticks**
```sql
CREATE TABLE ticks (
    "time" TIMESTAMPTZ NOT NULL,
    symbol TEXT NOT NULL,
    bid DECIMAL,
    ask DECIMAL,
    price DECIMAL,
    volume DECIMAL
);
SELECT create_hypertable('ticks', 'time');

CREATE INDEX ON ticks (symbol, "time" DESC);
```

**Important Notes**:
- Column name is `"time"` (with quotes) not `timestamp` - TimescaleDB convention
- Hypertable creation is REQUIRED for time-series optimization
- Index on (symbol, "time" DESC) enables fast queries by symbol and recent time
- All decimal fields can be NULL (bid/ask/price/volume might not always be present)

**Example SQL Commands**:
```sql
-- Insert tick data
INSERT INTO ticks ("time", symbol, bid, ask, price, volume)
VALUES ('2025-10-03T10:30:00Z', 'BTCUSD', 45000.50, 45001.00, 45000.75, 1.5);

-- Query recent ticks
SELECT * FROM ticks WHERE symbol = 'BTCUSD' ORDER BY "time" DESC LIMIT 10;
```

### Coding Standards
[Source: docs/architecture/phn-12-tiu-chun-code-coding-standards.md]

**CRITICAL - Must Follow:**
1. **NO direct environment variable access**: Use config package only
2. **NO hardcoded credentials**: Load TimescaleDB credentials from environment variables
3. **Structured logging ONLY**: Use `slog` package with JSON output
4. **Repository Pattern for database**: ALL TimescaleDB access MUST go through TickRepository interface or TimescaleTickRepository
5. **Use context.Context in Go**: ALL I/O operations (database operations, queries) must use context.Context

**Code Formatting:**
- Follow `go fmt` for all Go code
- Test files placed next to source files (e.g., timescale_repository.go → timescale_repository_test.go)

### Error Handling Strategy
[Source: docs/architecture/phn-11-chin-lc-x-l-li-error-handling-strategy.md]

**Logging**:
- Format: JSON
- Package: Use `slog` for Go (already setup in Story 1.2)
- Required log fields: timestamp, level, message, context data
- New events for Story 1.4:
  - `timescaledb_connected` - When connection to TimescaleDB succeeds
  - `timescaledb_connection_failed` - When connection fails (log error details)
  - `timescaledb_save_success` - When tick saved to TimescaleDB successfully
  - `timescaledb_save_failed` - When TimescaleDB insert operation fails (log error details)

**Retry Logic**:
- For Story 1.4: No retry logic for TimescaleDB writes (fail fast, log error, continue processing)
- Database connection pool should handle connection failures transparently
- Application should verify connection at startup and fail if TimescaleDB is not available

**Idempotency** (not required for Story 1.4 but awareness):
- Tick inserts are naturally append-only (no updates)
- Duplicate inserts are acceptable for MVP (deduplication can be Story 2.x concern)

### Go Dependencies to Add
- **PostgreSQL Driver**: `github.com/jackc/pgx/v5` (recommended) OR `github.com/lib/pq`
  - pgx is faster and more feature-rich
  - lib/pq is simpler and more standard
  - Recommend: Use pgx/v5 for better performance
- **Database Testing**: `github.com/testcontainers/testcontainers-go` (for integration tests)
- **SQL Mocking** (optional): `github.com/DATA-DOG/go-sqlmock` (for unit tests without real DB)

### Database Connection Notes
**Connection String Format** (for pgx or lib/pq):
```
postgres://user:password@host:port/dbname?sslmode=disable
```

**Example**:
```go
connStr := fmt.Sprintf("postgres://%s:%s@%s:%d/%s?sslmode=disable",
    cfg.TimescaleDBUser,
    cfg.TimescaleDBPassword,
    cfg.TimescaleDBHost,
    cfg.TimescaleDBPort,
    cfg.TimescaleDBName)
```

**Connection Pool Settings** (recommended):
- Max connections: 10-25 for single ingestion client
- Max idle connections: 5
- Connection lifetime: 1 hour
- Connection timeout: 5 seconds

```go

)

// Start TimescaleDB container
container, err := testcontainers.GenericContainer(ctx, testcontainers.GenericContainerRequest{
    ContainerRequest: testcontainers.ContainerRequest{
        Image:        "timescale/timescaledb:2.14.2-pg16",

    },

})
```
